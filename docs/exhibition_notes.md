# Gh0st in the L00p — exhibition notes

## Modes

### Video mode (current — use this)

Uses the optical flow morph video as background with scan line + text overlay on top.
Requires a morph video generated by `notebooks/pipeline.ipynb` on Colab.

```bash
# Export two portrait MP4s (one per screen)
python app.py --video data/<experiment_id>.mp4 --export

# Preview in OpenCV windows (press Q or ESC to quit)
python app.py --video data/<experiment_id>.mp4

# Override duration (default: read from sidecar JSON)
python app.py --video data/<experiment_id>.mp4 --export --duration 180

# Overwrite previous export (no timestamp suffix on output dir)
python app.py --video data/<experiment_id>.mp4 --export --overwrite
```

Output: `exports/exhibition/<experiment_id>_<timestamp>/screen_A.mp4` and `screen_B.mp4`

The sidecar JSON (`<experiment_id>.json`, placed next to the video) is read automatically
and provides duration, fps, and experiment ID. Without it, duration falls back to
`config.exhibition.proof_duration_seconds` (default: 72 s).

### Exhibition mode (original — still-image backed)

Uses still images from `faces_exhibition/` with the same overlay system.

```bash
python app.py --mode exhibition --export --duration 180
python app.py --mode exhibition
```

## Key files

- `exhibition/text_payload.json` — text pools for the terminal overlay
- `fonts/CourierPrime-Regular.ttf` — bundled font (used by overlay renderer)
- `config.py` — all tuneable parameters (`scan_cycle_seconds`, `clean_presentation`, etc.)

## Output

- Format: MP4 (H.264 via avc1 on macOS, mp4v fallback)
- Resolution: 1080×1920 (portrait)
- Frame rate: matches source video (typically 24 fps)

## Notes

- Screen A and Screen B receive the same background video but independent overlay
  state (text progression, corruption score).
- Hardware sync handled by venue (two Arduinos with looper function over LAN).
- `--overwrite` flag suppresses the timestamp suffix so re-exports land in the same
  folder, useful for iteration.
